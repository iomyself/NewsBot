# 版本记录更新

## V1.0.1 (2026-01-17)
**Status**: 已发布 / Released

### 新增功能 (New Features)
1.  **统一应用配置 (Centralized Config)**
    *   新增 `app_config.json` 配置文件。
    *   **filter_keywords**: 将关键词配置从代码中解耦，便于随时调整。
    *   **max_lookback_hours**: 新增时间回溯参数。

2.  **时间范围精准过滤 (Time Range Filtering)**
    *   重构日期解析逻辑，从单纯的字符串匹配升级为 `datetime` 对象比较。
    *   **逻辑**：仅抓取 `Current Time - Published Time <= max_lookback_hours` 的新闻。
    *   **优势**：避免抓取到虽是“今天”但实际是昨晚发布的旧闻（如果设置为24小时内），或支持抓取更久远的新闻（如48/72小时）。

## V1.0.0 (2026-01-17)
**Status**: 初始发布 / Initial Release

### 主要功能 (Core Features)

1.  **多源 RSS 新闻抓取**
    *   支持从 `rss_config.json` 读取无限数量的 RSS 订阅源。
    *   自动解析 Title, Link, Date 以及 **Description (新闻摘要)**。
    *   去除 HTML 标签，仅保留纯文本摘要。

2.  **智能关键词过滤**
    *   内置关键词过滤机制（当前配置：存款、理财、经济）。
    *   仅保留标题中包含指定关键词的新闻，减少无效信息干扰。

3.  **飞书多维表格集成 (Feishu/Lark Base)**
    *   **新闻同步**：将有效新闻自动写入飞书 "News" 数据表。
    *   **自动鉴权**：实现 `tenant_access_token` 的自动获取与本地缓存 (`token.json`)，避免频繁请求 API。
    *   **批量写入**：使用 `batch_create` 接口高效写入数据。

4.  **健壮的错误监控**
    *   **失效源记录**：自动捕获抓取失败的 RSS 源。
    *   **错误上报**：将失效源的 Title, URL 和具体错误信息 (ErrorMessage) 写入独立的 "Error Logs" 飞书数据表，便于维护。
    *   **独立配置**：支持为错误日志表配置独立的 App Token。

5.  **GitHub Actions 自动化部署**
    *   集成 `.github/workflows/daily_news.yml` 工作流。
    *   **定时运行**：每天北京时间 08:00 自动执行。
    *   **安全配置**：通过 GitHub Secrets 管理所有敏感凭证 (App ID, Secret, Tokens)。
